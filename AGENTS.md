# Agent instructions

## Committing

When you finish a piece of work, **commit it to the current branch — even if that
branch is `main`.** Don't leave finished work uncommitted, and don't create a new
branch just to avoid committing to `main`.

This deliberately overrides the usual "never commit to main" default. In this repo,
committing to the checked-out branch is the normal way to wrap up.

Do something else only when:

- **You're told to** — e.g. "don't commit", "open a PR", "work on a branch",
  "just stage it". Explicit instructions always win.
- **The work isn't actually done** — incomplete, or not yet verified. Finish it and
  check it first (typecheck / build / tests as appropriate for what you touched),
  then commit.

Keep each commit scoped to the work you did, with a clear message. **Don't push**
unless asked — committing locally is enough.
